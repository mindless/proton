Textile/Markdown support

Write articles in simple shorthand with Textile or Markdown. This will be magically converted to HTML.

index.haml

%div.article
  %h1
    Hello from Hyde via HAML!
  %p
    Lorem ipsum dolor sit
    amet, consecteteur...

Supervalid HTML via HAML

Write your HTML in terse, error-resistant HAML. HAML lets you generate HTML files from a simpler syntax.

css/style.scss

$red: #833;

.article {
  h1 { color: $red; }
  p  { line-height: 1.5; }
}

Painless CSS

You can use variables, nested rules, mixins, and other conveniences in your CSS files! Powered by Sass CSS.


↓ Build it

...and Hyde will build it for you.

$ hyde build



  • _output/foo.html
  • _output/index.html
  • _output/css/style.css
  • _output/images/background.png

After typing hyde build, Hyde will create the output folder for you. Anything that's not supported is simply copied over.

_output/foo.html

<div class='article'>
  <h1>Hello from Hyde via Textile!</h1>
  <p>Lorem <em>ipsum</em> dolor sit amet, consecteteur...</p>
</div>

Hyde automatically guesses file extensions. Textile files, for instance, become .html files...

_output/css/style.css

.article h1 {
  color: #833; }

.article p {
  line-height: 1.5; }

...and SCSS files become .css. You may specify your own extensions if need be.

Hyde works by taking a folder with files in any template format (Markdown, HAML, etc), and magically creating the output files (HTML, CSS, etc) for you.

Hyde takes this folder of files and crunches them.


↓ Add layouts

Adios, copy-paste. Hello, dynamic layouts.

_layouts/default.haml

%html
  %head

  %body

Step 1: Define a layout

Create a layout in _layouts/default.xxx.
it can be in any supported template language. ERB and HAML are popular choices.

+

index.erb

<h1>Welcome!</h1>
<p>Feel free to take a look around.</p>

Step 2: Create a page

Now your files will all use it.
→

_output/index.html

<html>
  <head>
    <title>index</title>
  </head>
  <body>
    <div id='container'>
      <h1>Welcome!</h1>
      <p>Feel free to take a look around.</p>
    </div>
  </body>
</html>

Gasp in amazement

Hyde will automatically put them together for you!

More than templates: it stores data.

posts/hello_world.textile

date: 2010-10-04
title: Hello there
layout: post
---

h3. Hello world!

This is my first post.

For any page, you may define metadata before the file begins. This lets you define what layout to use, the page title, or any arbitrary data. You may then use these in your code.

+

_layouts/post.haml

%article
  %h1= page.title
  %p.date= page.date.strftime('%Y %m %d')
  .content!= content

In your layouts, you may access the data through the page object.

→

_output/posts/hello_world.html

<article>
  <h1>Hello there</h1>
  <p class='date'>2010/10/04</p>
  <div class='content'>
    <h3>Hello world!</h3>
    <p>This is my first post.</p>
  </div>
</article>

Partials

index.erb

<%= partial :'post_author', 
    name: "Christian Troy" %>

Snippets getting repetitive? You may put stuff into "partials" that you can use without repeating the code.

+

_layouts/post_author.erb

<p class='colophon'>
  This is post was written
  by <%= post_author %>.
</p>

Simply define partials here.

→

_output/index.html

<p class='colophon'>
  This is post was written
  by Christian Troy.
</p>

Helpers

_extensions/helpers/helpers.rb

 module Hyde::Helpers
   def date_format(date)
     date.strftime('%Y/%m/%d')
   end
 end
Hyde supports extensions where you can add arbitrary functionality to your site. You may define helpers by adding an extension to your site.
+

index.erb

<footer>
  Generated on <%= date_format(Time.now) %>.
</footer>
→

_output/index.html

<footer>
  Generated on 2006/04/12.
</footer>
