Hyde

Hyde is the cleanest way to create static sites for any occasion.

Hyde works by taking a folder with files in any template format (Markdown, HAML, etc), and magically creating the output files (HTML, CSS, etc) for you.

Perfect for:

  • Static websites
  • HTML prototypes
  • Blogs
  • Project documentation

ls ./

./foo.textile
./index.haml
./css/style.scss
Take a folder with files in any template format.

./foo.textile

h1(hello). Hello from Hyde via textile!
Lorem *ipsum* dolor sit amet, consecteteur...
Write articles in simple shorthand with Textile or Markdown. This will be magically converted to HTML.

./index.haml

%div.article
  %h1 Hello from Hyde via HAML!
  %p Lorem ipsum dolor sit amet, consecteteur...
Write your HTML in terse, error-resistant HAML. HAML lets you generate HTML files from a simpler syntax.

./css/style.scss

$red: #833;

.article {
  h1 { color: $red; }
  p  { line-height: 1.5; }
}
You can use variables, nested rules, mixins, and other conveniences in your CSS files! Powered by Sass CSS.

Build

Hyde takes this a folder of these files and crunches

them.
hyde build

After typing hyde build, Hyde will create the output folder for you.

Layouts

You may define layouts for your files in any template language. Adios, tedious template copy-paste! You'd never have to repeat your HTML code again.

(You can even define per-page layouts, nested layouts, in any template language.)

Metadata

For any page, you may define metadata before the file begins. This lets you define what layout to use, the page title, or any arbitrary data. You may then use these in your code.

* posts/hello_world.textile + date: 2010-10-04 title: Hello there layout: post --- h3. Hello world! This is my first post. # This page has YAML metadata at the beginning # of the file. If you define `layout` (like above), # Hyde will use that layout instead of the default. * _layouts/post.haml = %article %h1= page.title %p.date= page.date.strftime('%Y %m %d') .content!= content # In your layouts, you may access the data through # the `page` object. * _output/posts/hello_world.html

Helpers

Hyde supports extensions where you can add arbitrary functionality to your site. You may define helpers by adding an extension to your site:

* /_extensions/helpers/helpers.rb module Hyde::Helpers def date_format(date) date.strftime('%Y/%m/%d') end end * /index.erb
